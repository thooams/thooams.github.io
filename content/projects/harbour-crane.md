### Manage easily Apps with Docker

<img src="/images/portfolio/harbour-crane/logo.png" alt="Harbour Crane logo" class="img-left" style="width: 200px;" />

As a web developer, I do not want to manage the applications on my server. I do not always control the techie, always problems with libraries sharing...

Fortunately, the container system has arrived and with it a number of solutions have emerged as [Core OS](https://coreos.com/), [Docker](https://www.docker.com/)... Since that day, managing apps on a server is much simpler. But it still lacks an overlay to facilitate the deployment of web applications through an interface without typing command lines.

And it is here that Harbour Crane's interesting.

### The big blue whale

<img src="/images/portfolio/harbour-crane/docker.jpg" alt="Docker" class="img-right" style="width: 200px;" />

We will not hide the fact, Docker revolutionized the world of containers. And to help it, Harbour Crane offers an application-oriented vision and fewer management of images or volumes.

The app uses Docker and Docker-Compose. Harbour Crane is used through a container. No installation, no library problems, no security issues.

## Interface

Web Interface for managing Apps

Apps, Images, Volumes can be managed by Harbour Crane interface.
{{< /section-dark >}}

{{< section-content >}}
### Start, Pause, Stop

With Harbour Crane, you can manage your applications with a file system construction made through a form. Once the application is created in the software, you can manage its start, its stop or its pause. If the server were to restart, Harbour Crane would run automatically each container according to its previous state.

### Databases shared

Databases can be shared and can be used when creating an application.

### Proxy

A proxy is present and configured to manage all containers running on the server.
